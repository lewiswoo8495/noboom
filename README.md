# noboom
不要抽到炸弹

卡牌类游戏
一、先描述静态的框架
如：基于元素：
（1）牌：总共有多少张牌、有哪些类型的卡牌（先不用描述效果）、各种类型的牌分别有多少张
（2）用户：多少个用户进行游戏（网络）、游戏币、用户游戏状态（正常手操、出局、托管）
（3）前置进入页面：自动匹配、进入游戏房间、创建游戏房间、开始游戏
（4）游戏页面的布局：如圆形桌、5人围坐、有转向的箭头、回合倒计时、抓牌按钮、出牌按钮、出牌区
（5）其他名词：底牌、手牌

## 本地运行

1. 安装依赖：

```bash
npm install
```

2. 启动后端：

```bash
npm run backend
```

3. 启动前端：

```bash
npm run dev
```

4. 浏览器打开 `http://localhost:5173`

## 已实现内容

- React 版游戏入口页
- `自动匹配`、`创建游戏房间`、`进入游戏房间` 按钮
- 点击 `进入游戏房间` 显示房间号输入和 `开始游戏` 按钮
- 简单后端房间逻辑：自动匹配、创建房间、加入房间、开始游戏
